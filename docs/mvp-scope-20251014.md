# MVPスコープ凍結（確定） — 2025-10-14

このドキュメントは、10/30 カットオーバーに向けた最小実装範囲（MVP）の確定版です。

## ゴール（例え話）
- フロア案内＝メニュー、通り名＝URL を先に固定し、最短で“開店”できる売り場だけを開ける（Must）。
- Should は拡張、Won’t は今回は見送り。

## 対象ページ（フロア一覧）
- Must: トップ / カテゴリ一覧 / 記事詳細 / 検索 / サイトマップ / 固定（About/Contact/Policy/Terms）
- Should: 特集一覧 / タグ一覧 / 人気記事ページ
- Won’t: 会員・コメント / AMP / PWA / 多言語

## 機能スコープ（売り場の設備）
- Must:
  - グローバルメニュー（6カテゴリ）、パンくず、検索（シンプル）
  - 301リダイレクト（旧→新の代表ルート）、ads.txt、基本SEO（title/desc/OG）、GA4 PV
  - サイトマップ/robots、自動OG画像（最低限）
- Should:
  - 人気/新着ブロック、タグクラウド
  - カテゴリの並び替え（新着/人気）・期間（全期間/30日）
- Won’t:
  - 会員・コメント、ブックマーク機能
  - 汎用CMS外の特別ワークフロー、AMP、PWA、多言語

## URL / IA（住所のルール）
- /（トップ）
- /{category}
- /{category}/{slug}
- /search?q=
- /site-map
- /about, /contact, /policy, /terms

## カテゴリ（フロア案内）とメニュー順
- カテゴリ: お金・家計管理（money） / 子育て・教育（parenting） / 暮らし・家事（life） / 働き方・キャリア（work） / 心と健康（health） / 特集（feature）
- メニュー順（PC/モバイル共通）: money / parenting / life / work / health / feature

## 非機能（安心・安全の基準）
- パフォーマンス: LCP < 2.5s（主要ページ）。画像は Sanity/CDN 最適化。
- アクセシビリティ: 見出し階層 / alt / フォーカス / コントラスト AA。
- セキュリティ: 基本ヘッダ（XCTO/Frame/Referrer/Permissions）。HSTS は本公開後に検討。
- 運用: .env と Sanity トークン管理、エラーログ確認動線。

## 受け入れ基準（デモシナリオ）
- 旧URL の代表 3 件が新URL で最終 200 に到達（301 → 200）。
- トップ → カテゴリ → 記事の導線が PC/スマホで崩れない。
- 検索 → 結果 → 記事が機能し、ads.txt が本番ドメインで 200。

## 依存/前提
- Sanity（projectId/dataset）と環境変数の設定。
- Vercel Domains = Ready。
